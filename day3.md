Today's lesson: Role-Based Prompting.
Same question. Same AI. Wildly different answers — just because I told Claude who to be first.
❌ Normal prompt:
"How should I handle errors in my API?"
→ Generic answer: "Use try-catch blocks and return proper status codes."
✅ Engineered prompt (with role):
"You are a senior backend engineer at a fintech company reviewing a junior dev's PR. How should I handle errors in my API?"
→ Expert-level answer: covered error classification, structured logging, retry-safe idempotency, security risks of leaking stack traces, and even how this would get scored in a real code review.
Same question. Completely different depth.
I'm prepping for backend engineering interviews right now, and this one shift — giving Claude a role before a task — has changed how I use it for everything: debugging, system design practice, even mock interview prep.
